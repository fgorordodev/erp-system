# ADR 0003: Role-based access control with explicit permissions

- Status: Accepted
- Date: 2026-07-23

## Context

ERP users need different administrative and operational capabilities. Role-only checks become too coarse as modules grow.

## Decision

Assign one role to each user and associate roles with explicit permissions through a join table. Resolve permissions during JWT authentication and enforce them with global guards and route metadata.

## Consequences

- controllers declare authorization requirements clearly
- authorization checks avoid repeated database calls within a request
- permission changes take effect on the next authenticated request because the strategy reloads the user
- future organization-scoped authorization will require extending the model
