# ADR 0003: Multi-role access control with explicit permissions

- Status: Accepted
- Date: 2026-07-23
- Updated: 2026-07-24 to reflect the implemented multi-role model

## Context

ERP users may need capabilities from more than one operational role. Role-only checks also become too coarse as modules grow.

## Decision

Assign users to one or more roles through `UserRole`, associate roles with explicit permissions through `RolePermission`, resolve current roles and permissions during JWT authentication, and enforce them with global guards plus route metadata.

## Consequences

- controllers declare authorization requirements clearly;
- one user may combine capabilities from multiple roles;
- permissions are deduplicated for the authenticated request;
- permission changes take effect on the next request because the strategy reloads persistence state;
- organization-scoped authorization will require extending assignments with tenant context.
