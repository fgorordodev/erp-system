# API conventions

## Base URLs

| Resource | Default URL |
|---|---|
| API | `http://localhost:3000/api` |
| Swagger UI | `http://localhost:3000/docs` |
| Health | `GET /api/health` |

## Authentication

Protected routes expect:

```http
Authorization: Bearer <access-token>
```

The OpenAPI security scheme is named `access-token`.

## Successful response envelope

```json
{
  "success": true,
  "data": {},
  "timestamp": "2026-07-23T20:00:00.000Z",
  "path": "/api/users"
}
```

## Error response envelope

```json
{
  "success": false,
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "User not found",
    "statusCode": 404
  },
  "timestamp": "2026-07-23T20:00:00.000Z",
  "path": "/api/users/example"
}
```

## Error categories

### Domain errors

Generated intentionally by modules:

- `USER_EMAIL_EXISTS`
- `USER_NOT_FOUND`
- future authentication and business errors

### Persistence fallback errors

Generated by the global Prisma filter for unhandled database failures:

- `UNIQUE_CONSTRAINT_VIOLATION`
- `FOREIGN_KEY_CONSTRAINT_VIOLATION`
- `RESOURCE_NOT_FOUND`
- `DATABASE_ERROR`

### Generic errors

- `VALIDATION_ERROR`
- `UNAUTHORIZED`
- `INTERNAL_ERROR`

## Validation

The global validation pipe uses:

- `whitelist: true`
- `forbidNonWhitelisted: true`
- `transform: true`
- implicit primitive conversion

Unknown request properties result in HTTP 400 rather than being silently accepted.

## Current endpoints

### Health

| Method | Route | Access | Description |
|---|---|---|---|
| GET | `/api/health` | Public | API and database health |

### Users

| Method | Route | Permission | Description |
|---|---|---|---|
| POST | `/api/users` | `USER_CREATE` | Create a user |
| GET | `/api/users` | `USER_READ` | List non-deleted users |
| GET | `/api/users/:id` | `USER_READ` | Get a non-deleted user |
| PATCH | `/api/users/:id` | `USER_UPDATE` | Update profile fields |
| PATCH | `/api/users/:id/status` | `USER_UPDATE` | Activate or deactivate |
| DELETE | `/api/users/:id` | `USER_DELETE` | Soft-delete and deactivate |

Exact permission values are defined in `src/security/rbac/permissions.constants.ts`; Swagger is the executable endpoint reference.

## REST guidelines for new modules

- Use plural resource names.
- Prefer nouns over action verbs.
- Use PATCH for partial updates.
- Return 201 for creation, 200 for reads/updates and appropriate 4xx errors.
- Keep domain errors specific and persistence errors generic.
- Never expose hashes, secrets, internal stack traces or Prisma metadata.
- Add pagination before list endpoints can grow substantially.
